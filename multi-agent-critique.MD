In this repo, read AGENTS.md and the production architecture docs first. Then run a multi-agent product critique:

1. Agent A: inventory every screen, major workflow, AI feature, backend dependency, and mismatch between docs and implementation.
2. Agent B: research comparable open-source products such as Goose, LibreChat, Open WebUI, Dify, Chatwoot, Matrix/Element, Signal, Mattermost, n8n, Langflow, and Flowise. Compare their interaction patterns against this app.
3. Agent C: critique each screen from a user/product perspective. Identify what feels useful, confusing, too dashboard-like, too Outlook-like, or missing.
4. Agent D: challenge every suggestion. Remove bloat, classify each idea as must-have, should-have, nice-to-have, or not-needed.

Have the agents debate, then produce a final screen-by-screen recommendation list with:
- what to keep
- what to change
- what to add
- what to reject
- priority
- rationale
- local file references
- external product references

Do not edit code unless I explicitly ask for implementation.
